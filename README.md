## About Me

프론트엔드 개발을 하면서 가장 크게 배우고 있는 건  
화면을 만드는 일은 결국 **사람의 경험을 설계하는 일**이라는 점입니다.

저는 React와 TypeScript를 중심으로  
실시간 웹 서비스, 상태 관리, 인증 흐름, 테스트 자동화까지 경험하며  
단순히 동작하는 화면이 아니라 **흐름이 자연스럽고, 함께 유지보수하기 쉬운 서비스**를 만드는 데 관심을 가져왔습니다.

혼자 빠르게 만드는 것보다,  
함께 읽고 이해하고 확장할 수 있는 구조를 만드는 개발자가 되고 싶습니다.

## Projects

### 🎮 MARBLE POP
**로비 - 대기방 - 게임 진행을 실시간으로 연결한 멀티플레이 웹 보드게임 서비스**

> 실시간으로 연결되는 서비스에서  
> 사용자 흐름이 끊기지 않도록 만드는 과정에 가장 많은 고민을 담은 프로젝트입니다.

**Tech**  
React · TypeScript · Vite · Tailwind CSS · React Router · Zustand · Socket.IO · MSW · Vitest · Playwright

**What I Did**
- 게임 런타임과 소켓 통신 구조를 담당했습니다.
- 실시간 게임 상태 반영 흐름을 설계하고, 재접속과 지연 상황에서도 안정적으로 동기화될 수 있도록 구현했습니다.
- mock socket 환경과 테스트 환경을 구성해 주요 사용자 흐름을 반복 검증할 수 있도록 만들었습니다.
- 라우팅, 상태 변화, 이벤트 흐름이 자연스럽게 이어지도록 구조를 정리했습니다.

🔗 [Repository](https://github.com/modoo-marble-team/Blue_Marble-frontend)  
🔗 [Live Demo](https://blue-marble-frontend.vercel.app/lobby)

### 📊 J.J JobHunt Dashboard
**취업 준비 과정을 한 화면에서 관리할 수 있도록 만든 대시보드 서비스**

> 해야 할 일, 지원 현황, 이력서, 메모를  
> 흩어지지 않게 한곳에서 관리할 수 있으면 좋겠다는 생각에서 시작한 프로젝트입니다.

**Tech**  
React · TypeScript · Vite · Tailwind CSS · React Router · Supabase · GitHub

**What I Did**
- 주요 화면 라우팅과 공통 레이아웃 구조를 설계했습니다.
- Context API와 useReducer 기반으로 지원 현황, 메모, 인증 상태를 전역 상태로 관리했습니다.
- localStorage 연동, 이력서 버전 관리, 메모 CRUD, 지원 통계 기능을 구현했습니다.
- Supabase Auth를 활용해 로그인, 회원가입, 세션 유지 흐름을 구성했습니다.

🔗 [Repository](https://github.com/woojaemin0106/J.J-JobHunt-Dashboard)  
🔗 [Live Demo](https://j-j-job-hunt-dashboard.vercel.app/)
